# E-commerce-Customer-churn
## 📌 Project Overview

This project focuses on analyzing customer churn behavior in an e-commerce platform using SQL. The dataset contains customer demographics, purchase behavior, engagement metrics, and satisfaction levels to identify patterns that lead to customer churn.

The goal is to extract meaningful insights that help businesses improve customer retention strategies.

## 🎯 Objective
Analyze factors influencing customer churn
Identify high-risk customers likely to churn
Understand customer behavior patterns
Provide data-driven insights for retention strategies

## ❗ Problem Statement

Customer churn is a major challenge in e-commerce businesses. Losing customers directly impacts revenue and growth.

This project aims to:

Detect key variables contributing to churn
Segment customers based on behavior and engagement
Support decision-making with actionable insights

## 🗂️ Dataset Description

The dataset is created using SQL and stored in a table named:

customer_churn

## 🔑 Key Columns:

- CustomerID 
- Churn(Customer churn status (1 = Churned, 0 = Active))
-Tenure 
- PreferredLoginDevice 
- CityTier 
- WarehouseToHome 
- PreferredPaymentMode
- Gender 
- HourSpendOnApp 
- NumberOfDeviceRegistered 
- PreferedOrderCat
- SatisfactionScore 
- MaritalStatus 
- NumberOfAddress
- Complain 
- OrderAmountHikeFromlastYear 
- CouponUsed 
- OrderCount 
- DaySinceLastOrder 
- CashbackAmount 

## 🛠️ Tools & Technologies

- SQL (MySQL) – Data creation and querying
- Database Management – Data storage and manipulation

## 📌 Conclusion

This analysis highlights critical factors driving customer churn in e-commerce. By focusing on customer satisfaction, engagement, and service quality, businesses can significantly reduce churn and improve long-term profitability.
